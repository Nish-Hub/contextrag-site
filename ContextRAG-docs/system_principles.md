# ContextRAG System Principles

1. Context First

AI responses should always be grounded in project knowledge.

2. Retrieval Before Generation

Context must be retrieved before sending prompts to the LLM.

3. Event Driven Architecture

Knowledge ingestion should be asynchronous and scalable.

4. API First

The RAG service must be usable by UI, IDE plugins, CLI tools and agents.

5. Persistent Knowledge

Conversations should become reusable knowledge.
